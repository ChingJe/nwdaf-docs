## What is free5GC?

free5GC is an open source project that develops and maintains an implementation of the 5G Core Network (5GC). The project was developed by a team led by Professor Jyh-Cheng Chen at Taiwan's National Chiao Tung University (now National Yang Ming Chiao Tung University).

Its primary goal is to provide a functional and standards-compliant 5GC that can be used by researchers, students, and developers for academic research, prototyping, and testing of 5G technologies and applications.

To understand what free5GC actually does, it helps to know what the 5G Core is responsible for. At the highest level, the core has two big jobs:

1. **Control Plane** → makes the decisions and sets the rules.
2. **Data Plane** → carries the user’s traffic through the network.

free5GC implements both planes according to 3GPP standards, providing a reliable foundation for experimentation and innovation.

Beyond these core functions, several characteristics make free5GC especially valuable to the research and development community.

### Written in Go

free5GC is developed in the Go programming language. Go was chosen because it handles concurrency and networking efficiently, which are critical for simulating how different parts of the 5G Core interact.

### Reference Implementation

The project closely follows 3GPP standards, serving as a reference implementation. This means its functions align with real-world 5G Core deployments, allowing researchers and developers to test ideas with confidence.

### Project Goal

free5GC was created to make a working, standards-compliant 5G Core available to the community. It lowers barriers for researchers, educators, and innovators to learn, experiment, and build prototypes without relying on proprietary systems.

free5GC is not built to compete with commercial telecom solutions. Its mission is to make 5G accessible for research and innovation.

## Origins of free5GC

In the 4G era, mobile core networks were tightly bound to proprietary hardware and vendor-specific software. This created major barriers for universities, startups, and smaller research teams, who often lacked the resources to access or experiment with these systems.

free5GC was created to break down those barriers by moving the mobile core into software and virtualization. This made the 5G Core something that could be studied, tested, and improved by anyone with a server and curiosity.

So what does this shift actually look like in practice? Let’s compare the closed 4G model with the open, flexible approach made possible by free5GC.

### 4G Proprietary Core vs. 5G with free5GC

| 4G Proprietary Core        | 5G with free5GC                               |
| -------------------------- | --------------------------------------------- |
| Locked to vendor hardware  | Runs on commodity servers or virtual machines |
| Expensive licensing        | Free and open source                          |
| Limited to large operators | Accessible to researchers and students        |
| Slow adoption of new ideas | Flexible for experimentation and prototyping  |

This transformation did not happen overnight. Before launching free5GC, Professor Jyh-Cheng Chen and his team carried out years of research into how core networks could become more modular and adaptable. Two key projects, RECO and SLV, laid the groundwork for what free5GC would eventually become.

### RECO (Reconfigurable Core Network)

RECO was a research effort that explored how a core network could be dynamically reconfigured to adapt to different services. While still a prototype, it highlighted the potential of modularity in the mobile core and pointed toward a more flexible future design.

### SLV (Service Level Virtualization)

SLV went a step further by breaking the core network into service-level blocks instead of treating it as one large node. This made it possible to manage different kinds of services separately, such as Human Service and IoT Service. SLV demonstrated how a service-based approach could make networks more flexible and customizable, anticipating the direction 5G would take.

Building on this foundation, Professor Jyh-Cheng Chen led a team of faculty and students at National Chiao Tung University to release the first generation of free5GC in January 2019. This initial version implemented a Non-Standalone (NSA) 5G core network and became the world's first open source 5G core network project. By October 2019, free5GC officially added support for the Standalone (SA) 5G Core.

The first release of free5GC opened the door for universities and research labs worldwide to experiment with a working 5G Core for the very first time.

## Evolution of free5GC

free5GC’s journey shows how quickly an academic project can grow into a community-driven initiative. From its first release in 2019 to its present-day Linux Foundation governance, the project has steadily expanded its scope and impact.

### Phase 1: First Release (January 2019)

The first version of free5GC supported the Non-Standalone (NSA) 5G Core, giving researchers hands-on access to a working implementation. This marked the world’s first open source 5G Core project.

### Phase 2: Breakthrough (October 2019)

Support for the Standalone (SA) Core was added, making free5GC the only open source project to provide both NSA and SA capabilities. This positioned it as a reference point for the wider 5G community.

### Phase 3: Expansion (2020–2021)

New 3GPP features were introduced, including:

* Convergent Charging
* Non-3GPP Access
* Traffic Influence
* Quality of Service (QoS) support

These additions made the project increasingly useful for realistic testing and prototyping.

### Phase 4: Collaboration (2021–Present)

Governance of free5GC transitioned to the Linux Foundation, with National Yang Ming Chiao Tung University (NYCU) serving as the primary contributor. The project now benefits from contributions by a growing open source community, detailed in the `CONTRIBUTORS.md` file.

Together, these milestones show how free5GC moved from a single university release to a project recognized and supported worldwide. Each stage added not only new technical capabilities but also greater accessibility for researchers, educators, and innovators working with 5G.

free5GC’s evolution reflects its dual identity as both a reference implementation for research and a global community effort shaping the future of 5G experimentation.

## How free5GC Fits in the 5G Landscape

free5GC is more than just a 5G Core implementation. It fills a critical gap between academic research, prototyping, and real-world network deployment.

While most commercial 5G Core solutions are tightly coupled with proprietary infrastructure and complex licensing models, free5GC provides a fully open and standards-compliant alternative. It supports both Non-Standalone (NSA) and Standalone (SA) modes and is actively maintained to align with evolving 3GPP specifications.

### Researchers

free5GC offers low-cost access to a working 5G Core, making it ideal for experimentation and validation of new networking ideas.

### Educators

Instructors and lab coordinators can use free5GC to demonstrate 5G Core principles in a classroom or lab setting.

### Startups and Small Companies

For teams building Proof of Concept (PoC) systems, free5GC provides a lightweight, standards-compliant core without the cost of commercial tools.

### System Integrators and Hobbyists

Developers working on custom testbeds or simulations can use free5GC as a reference implementation that mirrors real-world network functions.

By reducing the entry barrier to 5G core network development, free5GC helps broaden access to next-generation networking. It allows innovation that might otherwise be limited to organizations with commercial licenses or enterprise-scale infrastructure.

free5GC bridges the gap between research and deployment, giving learners, educators, and innovators an open platform to explore the possibilities of 5G.
