# 6.1.9 Features supported by the NFManagement service


The syntax of the supportedFeatures attribute is defined in clause 5.2.2 of 3GPP TS 29.571 \[7\].

The following features are defined for the Nnrf_NFManagement service.

Table 6.1.9-1: Features of supportedFeatures attribute used by Nnrf_NFManagement service

<td>Feature</td>
<td>M/O</td>
<td>Description</td>
</tr>
<tr class="even">
<td>1</td>
<td>Service-Map</td>
<td>M</td>
<td>Support of defining in the profile of the NF Instance the list of NF Service Instances based on a map type (i.e. support of the "nfServiceList" attribute in NFProfile).</td>
</tr>
<tr class="odd">
<td>2</td>
<td>Empty-Objects-Nrf-Info</td>
<td>O</td>
<td><p>Support of receiving empty JSON objects as values in the servedxxxInfo/servedxxxInfoList map attributes of the NrfInfo data structure used by an NRF during registration into another NRF (see clause 6.1.6.2.31).</p>
<p>An NRF that supports registering into another NRF shall support this feature.</p></td>
</tr>
<tr class="even">
<td>3</td>
<td>Inter-Plmn-Fqdn</td>
<td>M</td>
<td><p>Support of receiving intra-PLMN notification of changes in the NFProfile/NFService containing the "interPlmnFqdn" attribute (see clauses 6.1.6.2.2 and 6.1.6.2.3).</p>
<p>The NRF shall not send intra-PLMN notifications containing the "interPlmnFqdn" attribute to subscribing NF Instances that don't support this feature.</p></td>
</tr>
<tr class="odd">

<td>NRFSET</td>
<td>O</td>
<td><p>Support of the NRF Set feature as defined in clause 5.2.2.1.</p>
<p>All the NRF service instances of an NRF supporting this feature shall support the NRFSET feature.</p></td>
</tr>
<tr class="even">
<td>5</td>
<td>Complete-Profile-Subscription</td>
<td>O</td>
<td>Support subscriptions to the complete NF Profile of NF Instances (including, e.g. the authorization attributes) and their notifications.</td>
</tr>
<tr class="odd">
<td>6</td>
<td>Allowed-ruleset</td>
<td>O</td>
<td>Support registering RuleSets in NF (Service) profile</td>
</tr>
<tr class="even">
<td>7</td>
<td>Canary-Release</td>
<td>O</td>
<td>Support of "CANARY_RELEASE" value for NFStatus and NFServiceStatus, used for e.g. canary testing</td>
</tr>
<tr class="odd">
<td>8</td>
<td>DNN-List-Optimization</td>
<td>O</td>
<td><p>Support of dnnSmfInfoListId within SnssaiSmfInfoItem, and dnnUpfInfoListId within SnssaiUpfInfoItem</p>
<p>The NF consumer shall not make use of this feature if this feature is not supported by the NRF. See clause 5.2.2.2.2 on the NRF’s supported feature detection prior to registration.</p></td>
</tr>
<tr class="even">

<td>Shared-Data-Registration</td>
<td>O</td>
<td>If supported, NF consumers may register NFProfiles containing shared data IDs.</td>
</tr>
<tr class="odd">
<td>10</td>
<td>Shared-Data-Retrieval</td>
<td>O</td>
<td>If supported, NFProfiles sent to NF consumers may contain shared data IDs. Support of this feature also covers support of shared data retrieval, subscription, notification and un-subscription.</td>
</tr>
