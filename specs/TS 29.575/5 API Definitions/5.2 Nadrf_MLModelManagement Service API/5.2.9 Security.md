# 5.2.9 Security

As indicated in 3GPP TS 33.501 \[8\] and 3GPP TS 29.500 \[4\], the access to the Nadrf\_MLModelManagement API may be authorized by means of the OAuth2 protocol (see IETF RFC 6749 \[9\]), based on local configuration, using the "Client Credentials" authorization grant, where the NRF (see 3GPP TS 29.510 \[10\]) plays the role of the authorization server.

If OAuth2 is used, an NF Service Consumer, prior to consuming services offered by the Nadrf_MLModelManagement API, shall obtain a "token" from the authorization server, by invoking the Access Token Request service, as described in 3GPP TS 29.510 \[10\], clause 5.4.2.2.

NOTE: When multiple NRFs are deployed in a network, the NRF used as authorization server is the same NRF that the NF Service Consumer used for discovering the Nadrf_MLModelManagement service.

The Nadrf_MLModelManagement API defines a single scope "nadrf-mlmodelmanagement" for the entire service, and it does not define any additional scopes at resource or operation level.
