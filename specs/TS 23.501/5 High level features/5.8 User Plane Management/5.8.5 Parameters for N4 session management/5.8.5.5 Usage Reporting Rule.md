# 5.8.5.5 Usage Reporting Rule

The following table describes the Usage Reporting Rule (URR) that defines how a packet shall be accounted as well as when and how to report the measurements.

Table 5.8.5.5-1: Attributes within Usage Reporting Rule

<colgroup>
<col style="width: 27%" />
<col style="width: 42%" />
<col style="width: 30%" />
</colgroup>
<thead>
<tr class="header">
<th>Attribute</th>
<th>Description</th>
<th>Comment</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td>N4 Session ID</td>
<td>Identifies the N4 session associated to this URR</td>
<td></td>
</tr>
<tr class="even">
<td>Rule ID</td>
<td>Unique identifier to identify this information.</td>
<td>Used by UPF when reporting usage.</td>
</tr>
<tr class="odd">
<td>Reporting triggers</td>
<td>One or multiple of the events can be activated for the generation and reporting of the usage report.</td>
<td><p>Applicable events include:</p>
<p>- Start/stop of traffic detection with/without application instance identifier and deduced SDF filter reporting; Deletion of last PDR for a URR; Periodic measurement threshold reached; Volume/Time/Event measurement threshold reached; Immediate report requested; Measurement of incoming UL traffic; Measurement of discarded DL traffic; MAC address reporting in the UL traffic; unknown destination MAC/IP address; end marker packet has been received.</p></td>
</tr>
<tr class="even">
<td>Periodic measurement threshold</td>
<td>Defines the point in time for sending a periodic report for this URR (e.g. timeofday).</td>
<td><p>This allows generation of periodic usage report for e.g. offline charging.</p>
<p>It can also be used for realizing the Monitoring time of the usage monitoring feature.</p>
<p>It can also be used for realizing the Quota-Idle-Timeout, i.e. to enable the CP function to check whether any traffic has passed during this time.</p></td>
</tr>
<tr class="odd">
<td>Volume measurement threshold</td>
<td>Value in terms of uplink and/or downlink and/or total byte-count when the measurement report is to be generated.</td>
<td></td>
</tr>
<tr class="even">
<td>Time measurement threshold</td>
<td>Value in terms of the time duration (e.g. in seconds) when the measurement report is to be generated.</td>
<td></td>
</tr>
<tr class="odd">
<td>Event measurement threshold</td>
<td>Number of events (identified according to a locally configured policy) after which the measurement report is to be generated.</td>
<td></td>
</tr>
<tr class="even">
<td>Inactivity detection time</td>
<td>Defines the period of time after which the time measurement shall stop, if no packets are received.</td>
<td>Timer corresponding to this duration is restarted at the end of each transmitted packet.</td>
</tr>
<tr class="odd">
<td>Event based reporting</td>
<td>Points to a locally configured policy which is identifies event(s) trigger for generating usage report.</td>
<td></td>
</tr>
<tr class="even">
<td>Linked URR ID(s)</td>
<td>Points to one or more other URR ID.</td>
<td>This enables the generation of a combined Usage Report for this and other URRs by triggering their reporting. See clause 5.2.2.4, TS 29.244 [65].</td>
</tr>
<tr class="odd">
<td>Measurement Method</td>
<td>Indicates the method for measuring the network resources usage, i.e. the data volume, duration, combined volume/duration, or event.</td>
<td></td>
</tr>
<tr class="even">
<td>Measurement information</td>
<td>Indicates specific conditions to be applied for measurements</td>
<td><p>It is used to request:</p>
<p>- measurement before QoS enforcement and/or</p>
<p>- to pause or set to active a measurement as for the Pause of charging described in clause 4.4.4 and clause 4.23.14 of TS 23.502 [3] and/or</p>
<p>- to request reduced reporting for application start/stop events.</p></td>
</tr>
