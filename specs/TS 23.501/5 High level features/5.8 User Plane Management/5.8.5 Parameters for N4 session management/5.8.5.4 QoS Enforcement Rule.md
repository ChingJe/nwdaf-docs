# 5.8.5.4 QoS Enforcement Rule

The following table describes the QoS Enforcement Rule (QER) that defines how a packet shall be treated in terms of bit rate limitation and packet marking for QoS purposes. All Packet Detection Rules that refer to the same QER share the same QoS resources, e.g. MFBR.

Table 5.8.5.4-1: Attributes within QoS Enforcement Rule

<col style="width: 42%" />
<col style="width: 30%" />
</colgroup>

<tr class="header">
<th>Attribute</th>
<th>Description</th>
<th>Comment</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td>N4 Session ID</td>
<td>Identifies the N4 session associated to this QER</td>
<td></td>
</tr>
<tr class="even">
<td>Rule ID</td>
<td>Unique identifier to identify this information.</td>
<td></td>
</tr>
<tr class="odd">
<td>QoS Enforcement Rule correlation ID (NOTE 1)</td>
<td>An identity allowing the UP function to correlate multiple Sessions for the same UE and APN.</td>
<td>Is used to correlate QoS Enforcement Rules for APN-AMBR enforcement.</td>
</tr>
<tr class="even">
<td>Gate status UL/DL</td>
<td>Instructs the UP function to let the flow pass or to block the flow.</td>
<td>Values are: open, close, close after measurement report (for termination action "discard").</td>
</tr>
<tr class="odd">
<td>Maximum bitrate</td>
<td>The uplink/downlink maximum bitrate to be enforced for the packets.</td>
<td><p>This field may e.g. contain any one of:</p>
<p>- APN-AMBR (for a QER that is referenced by all relevant Packet Detection Rules of all PDN Connections to an APN) (NOTE 1).</p>
<p>- Session-AMBR (for a QER that is referenced by all relevant Packet Detection Rules of the PDU Session)</p>
<p>- QoS Flow MBR (for a QER that is referenced by all Packet Detection Rules of a QoS Flow)</p>
<p>- SDF MBR (for a QER that is referenced by the uplink/downlink Packet Detection Rule of a SDF)</p>
<p>- Bearer MBR (for a QER that is referenced by all relevant Packet Detection Rules of a bearer) (NOTE 1).</p></td>
</tr>
<tr class="even">
<td>Guaranteed bitrate</td>
<td>The uplink/downlink guaranteed bitrate authorized for the packets.</td>
<td><p>This field contains:</p>
<p>- QoS Flow GBR (for a QER that is referenced by all Packet Detection Rules of a QoS Flow)</p>
<p>- Bearer GBR (for a QER that is referenced by all relevant Packet Detection Rules of a bearer) (NOTE 1).</p></td>
</tr>
<tr class="odd">
<td>Averaging window</td>
<td>The time duration over which the Maximum and Guaranteed bitrate shall be calculated.</td>
<td>This is for counting the packets received during the time duration.</td>
</tr>
<tr class="even">
<td>Down-link flow level marking</td>
<td>Flow level packet marking in the downlink.</td>
<td>For UPF, this is for controlling the setting of the RQI in the encapsulation header as described in clause 5.7.5.3.</td>
</tr>
<tr class="odd">
<td>QoS Flow ID</td>
<td>QoS Flow ID to be inserted by the UPF.</td>
<td>The UPF inserts the QFI value in the tunnel header of outgoing packets.</td>
</tr>
<tr class="even">
<td>Paging Policy Indicator</td>
<td>Indicates the PPI value the UPF is required to insert in outgoing packets (see clause 5.4.3.2).</td>
<td>PPI applies only for DL traffic. The UPF inserts the PPI in the outer header of outgoing PDU.</td>
</tr>
<tr class="odd">
<td>Packet rate (NOTE 1)</td>
<td>Number of packets per time interval to be enforced.</td>
<td><p>This field contains any one of:</p>
<p>- downlink packet rate for Serving PLMN Rate Control (the QER is referenced by all PDRs of the UE belonging to PDN connections using CIoT EPS Optimisations as described in TS 23.401 [26]).</p>
<p>- uplink/downlink packet rate for APN Rate Control (the QER is referenced by all PDRs of the UE belonging to PDN connections to the same APN using CIoT EPS Optimisations as described in TS 23.401 [26]).</p></td>
</tr>
<tr class="even">
<td>End of Data Burst Marking Indication</td>
<td>Indicates to the UPF to provide an End of Data Burst indication of the last PDU of a Data burst to the NG-RAN over GTP-U</td>
<td>NG-RAN can configure UE power management schemes like connected mode DRX when UPF provides an indication of the End of Data Burst, see clause 5.37.8.3.</td>
</tr>
<tr class="odd">
<td>PDU Set Information marking Indicator</td>
<td>Indicates the UPF to insert PDU Set Information related to packets belonging to a PDU Set into GTP-U header.</td>
<td>UPF identifies PDU Sets in DL traffic and forwards PDU Set related information of each PDU to the NG-RAN over GTP-U, as described in clause 5.37.5.</td>
</tr>
<tr class="even">
<td>ECN marking for L4S indicator</td>
<td>Indicates the UPF to perform ECN marking for L4S for the corresponding QoS Flow.</td>
<td>UPF uses information sent by NG-RAN in GTP-U header extension to perform ECN marking for L4S for the corresponding direction.</td>
</tr>
<tr class="odd">
<td colspan="3">NOTE 1: This parameter is only used for interworking with EPC.</td>
</tr>
