# 7.2 Nnwdaf_AnalyticsSubscription Service

## 7.2.1 General

Service Description: This service enables the consumer to subscribe/unsubscribe for network data analytics and optionally its corresponding `Analytics Accuracy Information`.

When the subscription for network data analytics and optionally `Analytics Accuracy Information` is accepted by the `NWDAF` containing `AnLF`, the consumer receives from the `NWDAF` an identifier (`Subscription Correlation ID`) allowing to further manage (modify, delete) this subscription. The modification of Analytics and optionally `Analytics Accuracy Information` subscription can be enforced by `NWDAF` based on operator policy and configuration.

This service also enables the consumer to transfer analytics subscriptions to another `NWDAF` using the `Nnwdaf_AnalyticsSubscription_Transfer` service operation. Using this service operation, a consumer can request to the `NWDAF` to prepare for or take over analytics subscription(s), indicating also if and where analytics context is available.

## 7.2.2 Nnwdaf_AnalyticsSubscription_Subscribe service operation

Service operation name: `Nnwdaf_AnalyticsSubscription_Subscribe`

Description: Subscribes to `NWDAF` analytics and optionally `Analytics Accuracy Information` with specific parameters.

**Table 7.2.2-1: Nnwdaf_AnalyticsSubscription_Subscribe Inputs**

| Attribute Name | P | Description |
| :--- | :--- | :--- |
| (Set of) `Analytics ID(s)` | Required | As defined in Table 7.1-2 |
| `Target of Analytics Reporting` | Required | - |
| `Notification Target Address` (+ `Notification Correlation ID`) | Required | - |
| `Analytics Reporting Parameters` | Required | Including `Analytics target period`, etc. |
| `Analytics Filter Information` | Optional | - |
| `Time window for historical analytics` | Optional | - |
| `Subscription Correlation ID` | Optional | In the case of modification of the analytics subscription |
| `Preferred level of accuracy of the analytics` | Optional | - |
| `Preferred level of accuracy per analytics subset` | Optional | - |
| `Reporting Thresholds` | Optional | - |
| `Maximum number of objects requested` (`max`) | Optional | - |
| `Preferred order of results` | Optional | - |
| `Maximum number of SUPIs requested` (`SUPImax`) | Optional | - |
| `Time when analytics information is needed` | Optional | - |
| `Analytics Metadata Request` | Optional | - |
| (Set of) `NWDAF identifiers` | Optional | Used by the `NWDAF` service consumer when aggregating multiple analytic subscriptions |
| `Dataset Statistical Properties` | Optional | - |
| `Output strategy` | Optional | - |
| `Data time window` | Optional | - |
| `Consumer NF's serving area` or `NF ID` | Optional | - |
| `Information of previous analytics subscription` | Optional | `NWDAF identifier` (i.e. `Instance ID` or `Set ID`), `Analytics ID(s)` (including `SUPIs` and `Analytics Filter Information` for UE-related Analytics) and `Subscription Correlation ID` |
| `Use case context` | Optional | - |
| `Analytics Accuracy Request information` | Optional | - |
| `Analytics Feedback Information` | Optional | - |
| `QoS parameter set(s)` and corresponding value(s) | Optional | Optionally a requested `QoE` |
| `ADRF ID` | Optional | Where Analytics to be stored |
| `Storage Handling information` | Optional | As described in clause 5B.1 |

> **NOTE 1:** `Target of Analytics Reporting` can be provided per individual `Analytics ID`.
> 
> **NOTE 2:** `Analytics Filter Information`, `Reporting Thresholds`, maximum number of objects requested (`max`), maximum number of SUPIs requested (`SUPImax`), `Analytics Metadata Request`, `Dataset Statistical Properties`, `Output strategy`, `Data time window` and time when analytics information is needed can be provided per individual `Analytics ID`.
> 
> **NOTE 3:** `Analytics Feedback Information` only can be included in modification request for the existing analytics subscription.
> 
> **NOTE 4:** Other input parameters specific for different analytics ID can be also needed, as specified in the corresponding clause for each analytics ID.

**Table 7.2.2-2: Nnwdaf_AnalyticsSubscription_Subscribe Outputs**

| Attribute Name | P | Description |
| :--- | :--- | :--- |
| `Subscription Correlation ID` | Required | Required for management of this subscription when accepted |
| `Error response` | Required | When the subscription is not accepted |
| First corresponding `Analytics report` | Optional | Included if available and if analytics consumer requested immediate reporting |

> **NOTE 5:** When the `Target of Analytics Reporting` is a `SUPI` or a `GPSI` then the subscription may not be accepted, e.g. user consent is not granted and an error is sent to the consumer. When the `Target of Analytics Reporting` is an `Internal Group Id`, or a list of `SUPIs`/`GPSIs` or any UE, no error is sent, but a `SUPI` or `GPSI` is skipped if user consent is not granted.

## 7.2.4 Nnwdaf_AnalyticsSubscription_Notify service operation

Service operation name: `Nnwdaf_AnalyticsSubscription_Notify`

Description: `NWDAF` notifies the consumer instance of the analytics and optionally `Analytics Accuracy Information` that has subscribed to the specific `NWDAF` service. See also clause 6.1.3 for contents of the `Analytics Exposure`.

**Table 7.2.4-1: Nnwdaf_AnalyticsSubscription_Notify Inputs**

| Attribute Name | P | Description |
| :--- | :--- | :--- |
| `Notification Correlation Information` | Required | Indicates the `Notification Correlation ID` assigned by the consumer during subscription |
| Set of the tuple (`Analytics ID`, `Analytics specific parameters`) | Optional | Shall be present if output analytics are reported |
| `Timestamp of analytics generation` | Optional | Required when `ADRF` is deployed |
| `Validity period` | Optional | - |
| `Confidence` | Optional | - |
| `Revised waiting time` | Optional | - |
| `Analytics Metadata Information` | Optional | - |
| `Analytics Accuracy Information` | Optional | - |
| `Stop Analytics Output Consumption indication` | Optional | - |
| `Stop Analytics Output Consumption time window` | Optional | - |
| `Resume Analytics Output Consumption indication` | Optional | - |
| `Termination Request` | Optional | Indicates request to terminate with cause value (e.g. user consent revoked, `NWDAF` overload, etc.) |
| `Accuracy Information Termination` | Optional | - |
| `Subscription Change Notification Correlation ID` | Optional | Present if notification is for informing assignment of a new `Subscription Correlation Id`. Set to old ID. |
| `Subscription Correlation ID` | Optional | Present if notification is for informing assignment of a new `Subscription Correlation Id`. Set to new ID. |
| `Storage Approach` | Optional | As defined in clause 5B.1 |

> **NOTE 1:** `Validity period` can also be provided as part of `Analytics specific parameters` for some `NWDAF` output analytics.
> 
> **NOTE 2:** Other input parameters specific for different analytics ID can be also needed, as specified in the corresponding clause of each analytics ID.

**Table 7.2.4-2: Nnwdaf_AnalyticsSubscription_Notify Outputs**

| Attribute Name | P | Description |
| :--- | :--- | :--- |
| Operation execution result indication | Required | - |
