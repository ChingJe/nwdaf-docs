# 6.4.3 Output Analytics


The NWDAF services as defined in the clause 7.2 and 7.3 are used to expose the analytics.

\- Service Experience statistics information is defined in Table 6.4.3-1.

\- Service Experience predictions information is defined in Table 6.4.3-2.

Table 6.4.3-1: Service Experience statistics

<table>
<colgroup>
<col style="width: 25%" />
<col style="width: 74%" />
</colgroup>
<thead>
<tr class="header">
<th>Information</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td>Slice instance service experiences (0..max)</td>
<td>List of observed service experience information for each Network Slice instance.</td>
</tr>
<tr class="even">
<td>&gt; S-NSSAI</td>
<td>Identifies the Network Slice</td>
</tr>
<tr class="odd">
<td>&gt; NSI ID (NOTE 2)</td>
<td>Identifies the Network Slice instance within the Network Slice.</td>
</tr>
<tr class="even">
<td>&gt; Network Slice instance service experience</td>
<td>Service experience across Applications on a Network Slice instance over the Analytics target period (average, variance).</td>
</tr>
<tr class="odd">
<td>&gt; SUPI list (0..SUPImax) (NOTE 3)</td>
<td>List of SUPI(s) for which the slice instance service experience applies.</td>
</tr>
<tr class="even">
<td>&gt; Ratio (NOTE 3)</td>
<td>Estimated percentage of UEs with similar service experience (in the group, or among all UEs).</td>
</tr>
<tr class="odd">
<td>&gt; Spatial validity (NOTE 6)</td>
<td>Area where the Network Slice service experience analytics applies.</td>
</tr>
<tr class="even">
<td>&gt; Validity period</td>
<td>Validity period for the Network Slice service experience analytics as defined in clause 6.1.3.</td>
</tr>
<tr class="odd">
<td>Application service experiences (0..max)</td>
<td>List of observed service experience information for each Application.</td>
</tr>
<tr class="even">
<td>&gt; S-NSSAI</td>
<td>Identifies the Network Slice used to access the Application.</td>
</tr>
<tr class="odd">
<td>&gt; Application ID</td>
<td>Identification of the Application.</td>
</tr>
<tr class="even">
<td>&gt; Service Experience Type</td>
<td>Type of Service Experience analytics, e.g. on voice, video, other.</td>
</tr>
<tr class="odd">
<td>&gt; UE location (NOTE 1, NOTE 5)</td>
<td>Indicating the UE location information (e.g. TAI list, gNB ID, or location coordinates, etc) when the UE service is delivered.</td>
</tr>
<tr class="even">
<td>&gt; UPF Info (NOTE 4)</td>
<td>Indicating UPF serving the UE.</td>
</tr>
<tr class="odd">
<td>&gt; DNAI</td>
<td>Indicating which DNAI the UE service uses/camps on.</td>
</tr>
<tr class="even">
<td>&gt; DNN</td>
<td>DNN for the PDU Session which contains the QoS flow.</td>
</tr>
<tr class="odd">
<td>&gt; Application Server Instance Address</td>
<td>Identifies the Application Server Instance (IP address of the Application Server) or FQDN of Application Server.</td>
</tr>
<tr class="even">
<td>&gt; Service Experience</td>
<td>Service Experience over the Analytics target period (average, variance).</td>
</tr>
<tr class="odd">
<td>&gt; SUPI list (0..SUPImax) (NOTE 3)</td>
<td>List of SUPI(s) with the same application service experience.</td>
</tr>
<tr class="even">
<td>&gt; Ratio (NOTE 3)</td>
<td>Estimated percentage of UEs with similar service experience (in the group, or among all UEs).</td>
</tr>
<tr class="odd">
<td>&gt; Spatial validity (NOTE 6)</td>
<td>Area where the Application service experience analytics applies.</td>
</tr>
<tr class="even">
<td>&gt; Validity period</td>
<td>Validity period for the Application service experience analytics as defined in clause 6.1.3.</td>
</tr>
<tr class="odd">
<td><p>&gt; RAT Type</p>
<p>(NOTE 7)</p></td>
<td>Indicating the list of RAT type(s) for which the application service experience analytics applies.</td>
</tr>
<tr class="even">
<td><p>&gt; Frequency</p>
<p>(NOTE 7)</p></td>
<td>Indicating the list of carrier frequency value(s) of UE's serving cell(s) where the application service experience analytics applies.</td>
</tr>
<tr class="odd">
<td>&gt; SSC Mode</td>
<td>SSC Mode selected for the PDU Session used to associate with the application.</td>
</tr>
<tr class="even">
<td>&gt; PDU Session Type</td>
<td>Type of PDU Session used to associate with the application.</td>
</tr>
<tr class="odd">
<td>&gt; Access Type</td>
<td>List of Access Type(s) used for the PDU Session for the application.</td>
</tr>
<tr class="even">
<td colspan="2"><p>NOTE 1: This information element is an Analytics subset that can be used in "list of analytics subsets that are requested" and "Preferred level of accuracy per analytics subset".</p>
<p>NOTE 2: The NSI ID is an optional parameter. If not provided the Slice instance service experience indicates the service experience for the S-NSSAI.</p>
<p>NOTE 3: The SUPI list and Ratio in the service experience information for an application can be omitted, if the corresponding parameter(s) is/are provided and are assigned with the same value(s) in the service experience information for the slice instance which the application belongs to. Otherwise, the SUPI list and Ratio are mandatory to be provided for an application service experience.</p>
<p>NOTE 4: If the consumer NF is an AF, the "UPF info" shall not be included.</p>
<p>NOTE 5: When possible and applicable to the access type, UE location is provided according to the preferred granularity of location information. UE location shall only be included if the Consumer analytics request is for single UE or a list of UEs. Inclusion of UE location requires user consent.</p>
<p>NOTE 6: The Spatial validity is present in the output parameters if the consumer provided the Area of Interest as defined in Table 6.4.1-1.</p>
<p>NOTE 7: When "any" value has been provided in the request (e.g. "any" RAT type, "any" frequency, or "any" for all the RAT type and frequency indication), the NWDAF provides an instance of the Application service experience per combination of RAT Type(s) and/or Frequency value(s) having the same Service Experience.</p></td>
</tr>
</tbody>
</table>
