# 5.5.2 Simple Data Types


This clause specifies common simple data types.

Table 5.5.2-1: Simple Data Types

<table>
<colgroup>
<col style="width: 20%" />
<col style="width: 20%" />
<col style="width: 59%" />
</colgroup>
<tbody>
<tr class="odd">
<td>Type Name</td>
<td>Type Definition</td>
<td>Description</td>
</tr>
<tr class="even">
<td>Qfi</td>
<td>integer</td>
<td>Unsigned integer identifying a QoS flow, within the range 0 to 63.</td>
</tr>
<tr class="odd">
<td>QfiRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "Qfi" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>5Qi</td>
<td>integer</td>
<td>Unsigned integer representing a 5G QoS Identifier (see clause 5.7.2.1 of TS 23.501 [8]), within the range 0 to 255.</td>
</tr>
<tr class="odd">
<td>5QiRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "5Qi" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>BitRate</td>
<td>string</td>
<td><p>String representing a bit rate that shall be formatted as follows:</p>
<p>Pattern: '^\d+(\.\d+)? (bps|Kbps|Mbps|Gbps|Tbps)$'</p>
<p>(NOTE)</p>
<p>Examples:</p>
<p>"125 Mbps", "0.125 Gbps", "125000 Kbps"</p></td>
</tr>
<tr class="odd">
<td>BitRateRm</td>
<td>string</td>
<td>This data type is defined in the same way as the "BitRate" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>PacketRate</td>
<td>string</td>
<td><p>String representing a packet rate, i.e. packets per second, that shall be formatted as follows:</p>
<p>Pattern: '^\d+(\.\d+)? (pps|kpps|Mpps|Gpps|Tpps)$'</p>
<p>(NOTE)</p>
<p>Examples:</p>
<p>"125 Mpps", "0.125 Gpps", "125000 kpps"</p></td>
</tr>
<tr class="odd">
<td>PacketRateRm</td>
<td>string</td>
<td>This data type is defined in the same way as the "PacketRate" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>TrafficVolume</td>
<td>string</td>
<td><p>String representing a traffic volume measured in bytes that shall be formatted as follows:</p>
<p>Pattern: '^\d+(\.\d+)? (B|kB|MB|GB|TB)$'</p>
<p>(NOTE)</p>
<p>Examples:</p>
<p>"125 MB", "0.125 GB", "125000 kB"</p></td>
</tr>
<tr class="odd">
<td>TrafficVolumeRm</td>
<td>string</td>
<td>This data type is defined in the same way as the "TrafficVolume" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>ArpPriorityLevel</td>
<td>integer</td>
<td><p>Unsigned integer indicating the ARP Priority Level (see clause 5.7.2.2 of TS 23.501 [8]), within the range 1 to 15.</p>
<p>Values are ordered in decreasing order of priority, i.e. with 1 as the highest priority and 15 as the lowest priority.</p></td>
</tr>
<tr class="odd">
<td>ArpPriorityLevelRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "ArpPriorityLevel" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>5QiPriorityLevel</td>
<td>integer</td>
<td><p>Unsigned integer indicating the 5QI Priority Level (see clauses 5.7.3.3 and 5.7.4 of TS 23.501 [8]), within the range 1 to 127.</p>
<p>Values are ordered in decreasing order of priority, i.e. with 1 as the highest priority and 127 as the lowest priority.</p></td>
</tr>
<tr class="odd">
<td>5QiPriorityLevelRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "5QiPriorityLevel" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>PacketDelBudget</td>
<td>integer</td>
<td><p>Unsigned integer indicating Packet Delay Budget (see clauses 5.7.3.4 and 5.7.4 of TS 23.501 [8])), expressed in milliseconds.</p>
<p>Minimum = 1.</p></td>
</tr>
<tr class="odd">
<td>PacketDelBudgetRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "PacketDelBudget" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>PacketErrRate</td>
<td>string</td>
<td><p>String representing Packet Error Rate (see clause 5.7.3.5 and 5.7.4 of TS 23.501 [8]), expressed as a "<em>scalar</em> x 10-k" where the scalar and the <em>exponent k are each encoded as one decimal digit</em>.</p>
<p>Pattern: '^([0-9]E-[0-9])$'</p>
<p>Examples:</p>
<p>Packer Error Rate 4x10<sup>-6</sup> shall be encoded as "4E-6".</p>
<p>Packer Error Rate 10<sup>-2</sup> shall be encoded as "1E-2".</p></td>
</tr>
<tr class="odd">
<td>PacketErrRateRm</td>
<td>string</td>
<td>This data type is defined in the same way as the "PacketErrRate" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>PacketLossRate</td>
<td>integer</td>
<td><p>Unsigned integer indicating Packet Loss Rate (see clauses 5.7.2.8 and 5.7.4 of TS 23.501 [8]), expressed in tenth of percent.</p>
<p>Minimum = 0. Maximum = 1000.</p></td>
</tr>
<tr class="odd">
<td>PacketLossRateRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "PacketLossRate" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>AverWindow</td>
<td>integer</td>
<td><p>Unsigned integer indicating Averaging Window (see clause 5.7.3.6 and 5.7.4 of TS 23.501 [8]), expressed in milliseconds.</p>
<p>Minimum = 1. Maximum = 4095. Default = 2000..</p></td>
</tr>
<tr class="odd">
<td>AverWindowRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "AverWindow" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>MaxDataBurstVol</td>
<td>integer</td>
<td><p>Unsigned integer indicating Maximum Data Burst Volume (see clauses 5.7.3.7 and 5.7.4 of TS 23.501 [8]), expressed in Bytes.</p>
<p>Minimum = 1. Maximum = 4095.</p></td>
</tr>
<tr class="odd">
<td>MaxDataBurstVolRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "MaxDataBurstVol" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>SamplingRatio</td>
<td>integer</td>
<td><p>Unsigned integer indicating Sampling Ratio (see clauses 4.15.1 of TS 23.502 [28], expressed in percent.</p>
<p>Minimum = 1. Maximum = 100</p></td>
</tr>
<tr class="odd">
<td>SamplingRatioRM</td>
<td>integer</td>
<td>This data type is defined in the same way as the "SamplingRatio" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>RgWirelineCharacteristics</td>
<td>Bytes</td>
<td>RG Level Wireline Access Characteristics (see BBF TR-456 [41] and BBF TR-470 [37]). It shall be encoded as a string with format "byte" as defined in OpenAPI Specification [3], i.e. base64 encoded characters, representing the RG-Level Wireline Access Characteristics encoded as specified in clause 7.5 of BBF TR-470 [37].</td>
</tr>
<tr class="odd">
<td>RgWirelineCharacteristicsRm</td>
<td>Bytes</td>
<td>This data type is defined in the same way as the "RgWirelineCharacteristics" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>ExtMaxDataBurstVol</td>
<td>integer</td>
<td><p>Unsigned integer indicating Maximum Data Burst Volume (see clauses 5.7.3.7 and 5.7.4 of TS 23.501 [8]), expressed in Bytes.</p>
<p>Minimum = 4096. Maximum = 2000000.</p></td>
</tr>
<tr class="odd">
<td>ExtMaxDataBurstVolRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "ExtMaxDataBurstVol" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>ExtPacketDelBudget</td>
<td>integer</td>
<td><p>Unsigned integer indicating Packet Delay Budget (see clauses 5.7.3.4 and 5.7.4 of TS 23.501 [8])), expressed in 0.01 milliseconds.</p>
<p>Minimum = 1.</p></td>
</tr>
<tr class="odd">
<td>ExtPacketDelBudgetRm</td>
<td>integer</td>
<td>This data type is defined in the same way as the "ExtPacketDelBudget" data type, but with the OpenAPI "nullable: true" property.</td>
</tr>
<tr class="even">
<td>Metadata</td>
<td>string</td>
<td><p>This datatype contains information that is transparently passed to UPF and the UPF provides it to the service functions in N6-LAN.</p>
<p>When present, this IE shall be encoded as a string with format "byte" as defined in OpenAPI Specification [3], i.e. base64-encoded characters, representing the Metadata.</p></td>
</tr>
<tr class="odd">
<td colspan="3">NOTE: The prefixes used for bit rate unit "bps", packet rate unit "pps" and traffic volume in byte unit "B" shall be taken as x1000 multipliers and were meant to follow the standard symbols from "The International System of Units" (<a href="https://www.bipm.org/en/measurement-units/si-prefixes">https://www.bipm.org/en/measurement-units/si-prefixes</a>).<br />
However, even when the standard symbol for 10^3 multiplier is "k", in the present specification it has been defined as "K", and has been kept as such due to backwards-compatibility with earlier versions of this specification.</td>
</tr>
