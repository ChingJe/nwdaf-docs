<td>numericValNumUes</td>
<td>integer</td>
<td>C</td>
<td>0..1</td>
<td><p>This attribute may be present in the following cases:</p>
<p>- to configure the monitoring threshold for the reporting of the number of registered UEs for a network slice identified by an S-NSSAI;</p>
<p>- to report the network slice status for the current number of registered UEs.</p>
<p>When used to configure the monitoring threshold for an S-NSSAI, it shall contain the configured event monitoring threshold value for monitoring the number of registered UEs expressed in a numerical value.</p>
<p>When used to report the network slice status for an S-NSSAI, it shall contain the current number of registered UEs in the concerned network slice expressed in a numerical value.</p>
<p>For threshold based reporting and the threshold value for the number of registered UEs in the concerned network slice was previously configured in the form of a numerical value, this attribute shall contain the current number of registered UEs in the concerned network slice expressed in a numerical value.</p></td>
</tr>

<td>numericValNumPduSess</td>
<td>integer</td>
<td>C</td>
<td>0..1</td>
<td><p>This attribute may be present in the following cases:</p>
<p>- to configure the monitoring threshold for the reporting of the number established PDU session for a network slice identified by an S-NSSAI;</p>
<p>- to report the network slide status for the number of established PDU sessions.</p>
<p>When used to configure the monitoring threshold for an S-NSSAI, it shall contain the configured event monitoring threshold value for monitoring the number of established PDU sessions expressed in a numerical value.</p>
<p>When used to report the network slice status for an S-NSSAI, it shall contain the current number of established PDU sessions in the concerned network slice expressed in a numerical value.</p>
<p>For threshold based reporting and the threshold value for the number of established PDU sessions in the concerned network slice was previously configured in the form of a numerical value,, this attribute shall contain the current number of established PDU sessions in the concerned network slice expressed in a numerical value.</p></td>

<td>percValueNumUes</td>
<td>integer</td>
<td>C</td>
<td>0..1</td>
<td><p>This attribute may be present in the following cases:</p>
<p>- to configure the monitoring threshold for the reporting of the number of registered UEs for a network slice identified by an S-NSSAI;</p>
<p>- to report the network slice status for the number of registered UEs .</p>
<p>When used to configure the monitoring threshold for an S-NSSAI, it shall contain an unsigned integer indicating the event monitoring threshold value for the number of registered UEs expressed in percentage format.</p>
<p>When used to report the network slice status for an S-NSSAI, it shall contain the current number of registered UEs in the concerned network slice expressed as a percentage. For threshold based reporting and the threshold value for the number of registered UEs in the concerned network slice was previously configured as a percentage, this attribute shall contain the current number of registered UEs in the concerned network slice expressed as a percentage.</p>
<p>Minimum = 0. Maximum = 100.</p></td>

<td>percValueNumPduSess</td>
<td>integer</td>
<td>C</td>
<td>0..1</td>
<td><p>This IE may be present in the following cases:</p>
<p>- to configure the monitoring threshold for the reporting of the number of established PDU sessions for a network slice identified by an S-NSSAI;</p>
<p>- to report the network slice status for the number of established PDU sessions.</p>
<p>When used to configure the monitoring threshold for an S-NSSAI, it shall contain an unsigned integer indicating the event monitoring threshold value for the number of established PDU sessions expressed in percentage format.</p>
<p>When used to report the status of an S-NSSAI, it shall contain the current number of established PDU sessions in the concerned network slice expressed as a percentage.</p>
<p>For threshold based reporting and the threshold value for the number of established PDU sessions in the concerned network slice was previously configured as a percentage, this attribute shall contain the current number of established PDU sessions in the concerned network slice expressed as a percentage.</p>
<p>Minimum = 0. Maximum = 100.</p></td>
</tr>
<tr class="odd">
<td>uesWithPduSessionInd</td>
<td>boolean</td>
<td>C</td>
<td>0..1</td>
<td><p>This IE may be present if the numericValNumUes IE or the percValueNumUes IE is present, when reporting the network slice status for an S-NSSAI.</p>
<p>When present, it shall be set as follows:</p>
<p>- True: the numericValNumUes and percValueNumUes report the number of UEs with at least one PDU session/PDN connection.</p>
<p>- False (default): the numericValNumUes and percValueNumUes report the current number of registered UEs.</p></td>
