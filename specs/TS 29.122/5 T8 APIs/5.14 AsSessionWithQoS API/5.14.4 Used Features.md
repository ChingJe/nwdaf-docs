# 5.14.4 Used Features

The table below defines the features applicable to the AsSessionWithQoS API. Those features are negotiated as described in subclause 5.2.7.

Table 5.14.4-1: Features used by AsSessionWithQoS API

<td><strong>Feature</strong></td>
<td><strong>Description</strong></td>

<tr class="even">
<td>1</td>
<td>Notification_websocket</td>
<td>The delivery of notifications over Websocket is supported according to clause 5.2.5.4. This feature requires that the Notification_test_event featute is also supported.</td>
</tr>
<tr class="odd">
<td>2</td>
<td>Notification_test_event</td>
<td>The testing of notifications connections is supported according to clause 5.2.5.3.</td>

<tr class="even">

<td>EthAsSessionQoS_5G</td>
<td>Setting up required QoS for Ethernet UE. This feature may only be supported in 5G.</td>
</tr>
<tr class="odd">
<td>4</td>
<td>MacAddressRange_5G</td>
<td>Indicates the support of a set of MAC addresses with a specific range in the traffic filter. This feature may only be supported in 5G.</td>
</tr>
<tr class="even">
<td>5</td>
<td>AlternativeQoS_5G</td>
<td>Indicates the support of alternative QoS requirements and the QoS notification (i.e. whether the QoS targets for SDF(s) are not guaranteed or guaranteed again). This feature may only be supported in 5G.</td>
</tr>
<tr class="odd">
<td>6</td>
<td>QoSMonitoring_5G</td>
<td>Indicates the support of QoS Monitoring functionality and the report for packet delay monitoring. This feature may only be supported in 5G.</td>
</tr>
<tr class="even">
<td>7</td>
<td>DisableUENotification_5G</td>
<td>Indicates the support of disabling QoS flow parameters signalling to the UE when the SMF is notified by the NG-RAN of changes in the fulfilled QoS situation. This feature may only be supported in 5G. This feature requires that the AlternativeQoS_5G feature is also supported.</td>
</tr>
<tr class="odd">
<td>8</td>
<td>TSC_5G</td>
<td>Indicates the support of Time Sensitive Communication. This feature may only be supported in 5G.</td>
</tr>
<tr class="even">
<td>9</td>
<td>AppId</td>
<td>Indicates the support of dynamically providing the Application Identifier via the API.</td>
</tr>
<tr class="odd">
<td>10</td>
<td>ExposureToEAS</td>
<td>This feature indicates the support of direct notification in 5GC. This feature requires that the QoSMonitoring_5G feature is also supported.</td>
</tr>
<tr class="even">
<td>11</td>
<td>enNB</td>
<td>Indicates the support of enhancements to the northbound interfaces.</td>
</tr>
<tr class="odd">
<td>12</td>
<td>AltQosWithIndParams_5G</td>
<td>This feature indicates the support of provisioning Alternative Service Requirements with individual QoS parameters. This feature requires that the AlternativeQoS_5G feature is also supported.</td>
</tr>
<tr class="even">
<td>13</td>
<td>EnEthAsSessionQoS_5G</td>
<td>Indicates the support of required QoS for Ethernet UE, allowing to indicate separately different UL and/or DL Ethernet flows. This feature may only be supported in 5G.</td>
</tr>
<tr class="odd">
<td>14</td>
<td>enNB_5G</td>
<td>Indicates the support of enhancements to the northbound interfaces and only applicable to 5G.</td>
</tr>
<tr class="even">
<td>15</td>
<td>PacketDelayFailureReport</td>
<td>Indicates the support of packet delay failure report as part of QoS Monitoring procedures. This feature requires that QoSMonitoring_5G is supported. This feature may only be supported in 5G.</td>
</tr>
<tr class="odd">
<td>16</td>
<td>ToSTC_5G</td>
<td>Indicates the support of Type of Service or Traffic Class. This feature may only be supported in 5G.</td>
</tr>
<tr class="even">
<td>17</td>
<td>EnTSCAC</td>
<td><p>Indicates the support of extensions to TSCAC and the RAN feedback for BAT offset and adjusted periodicity.</p>
<p>This feature may only be supported in 5G, and requires that the TSC_5G feature is also supported.</p></td>
</tr>
<tr class="odd">
<td>18</td>
<td>AltQoSProfilesSupportReport</td>
<td>This feature indicates the support of the report of whether Alternative QoS parameters are supported by the access network. This feature requires that AlternativeQoS_5G and/or AltQosWithIndParams_5G features are also supported.</td>
</tr>
<tr class="even">
<td>19</td>
<td>ExtQoS_5G</td>
<td>This feature indicates the support of extended QoS parameters. This feature may only be supported in 5G.</td>
</tr>
<tr class="odd">
<td>20</td>
<td>MultiMedia</td>
<td>Indicates the support for multi-modal or multimedia flows for single UE and multiple UE. This feature may only be supported in 5G. This feature may be used in eXtend Reality (XR) use cases.</td>
</tr>
<tr class="even">
<td>21</td>
<td>ExtErrors</td>
<td>Indicates the support of additional application errors related to authorization or PDU Session availability.</td>
</tr>
<tr class="odd">
<td>22</td>
<td>QoSTiming_5G</td>
<td>This feature indicates the support of QoS timing information for the transfer and support of data transmission (e.g., AI/ML transmission). This feature may only be supported in 5G.</td>
</tr>
<tr class="even">
<td>23</td>
<td>ListUE_5G</td>
<td>Indicates the support for the list of UEs This feature may only be supported in 5G.</td>
</tr>
<tr class="odd">
<td>24</td>
<td>GMEC</td>
<td><p>This feature indicates the support of Generic Group Management Exposure and Communication related enhancements.</p>
<p>The following functionalities are supported:</p>
<p>- Support AF requested QoS for a UE or group of UE(s) not identified by the UE address(es).</p>
<p>This feature may only be supported in 5G.</p>
<p>This feature requires the support of the "QoSMonitoring_5G" and "AltQosWithIndParams_5G" features.</p></td>
</tr>
<tr class="even">
<td>25</td>
<td>PDUSetHandling</td>
<td><p>This feature indicates the support of PDU Set handling. This feature may be used for eXtended Reality (XR) and interactive media services.</p>
<p>This feature may only be supported in 5G.</p></td>
</tr>
<tr class="odd">
<td>26</td>
<td>RTLatency</td>
<td><p>This feature indicates the support of Round-Trip latency. This feature may be used for eXtended Reality (XR) and interactive media services.</p>
<p>This feature may only be supported in 5G.</p></td>
</tr>
<tr class="even">
<td>27</td>
<td>EnQoSMon</td>
<td><p>This feature indicates the support of enhanced QoS monitoring functionality, i.e. the enhancement of packet delay QoS monitoring, and/or, the report of the congestion information, and/or, the RTT delay over two QoS flows, and/or, the data rate information, and/or, the Packet Delay Variation monitoring.</p>
<p>This feature requires that QoSMonitoring_5G is supported.</p>
<p>In order to support the report of packet delay measurement failure, the PacketDelayFailureReport feature also requires to be supported.</p>
<p>This feature may only be supported in 5G.</p></td>
</tr>
<tr class="odd">
<td>28</td>
<td>PowerSaving</td>
<td><p>This feature indicates the support of the Power Saving for different traffic measurement<strong>.</strong></p>
<p>This feature may only be supported in 5G.</p></td>
</tr>
<tr class="even">
<td>29</td>
<td>L4S</td>
<td><p>This feature indicates the support of the AF indication of ECN marking for L4S support.</p>
<p>This feature may only be supported in 5G.</p></td>
</tr>
