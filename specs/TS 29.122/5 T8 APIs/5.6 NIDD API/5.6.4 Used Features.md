# 5.6.4 Used Features

The table below defines the features applicable to the NIDD API. Those features are negotiated as described in clause 5.2.7.

Table 5.6.4-1: Features used by NIDD API

<colgroup>
<col style="width: 9%" />
<col style="width: 32%" />

<td>Feature</td>
<td>Description</td>
</tr>
<tr class="even">
<td>1</td>
<td>GroupMessageDelivery</td>
<td>This feature indicates the support of group message delivery via MT NIDD as defined in clause 5.5.3 of 3GPP TS 23.682 [2].</td>
</tr>
<tr class="odd">
<td>2</td>
<td>Notification_websocket</td>
<td>The delivery of notifications over Websocket is supported according to clause 5.2.5.4. This feature requires that the Notification_test_event featute is also supported.</td>
</tr>
<tr class="even">
<td>3</td>
<td>Notification_test_event</td>
<td>The testing of notification connection is supported according to clause 5.2.5.3.</td>
</tr>
<tr class="odd">
<td>4</td>
<td>MT_NIDD_modification_cancellation</td>
<td>Modification and cancellation of an individual MT NIDD resource.</td>
</tr>
<tr class="even">
<td>5</td>
<td>Rds_port_verification</td>
<td>This feature indicates the support of RDS port verification in the MO/MT NIDD delivery.</td>
</tr>
<tr class="odd">
<td>6</td>
<td>Rds_dynamic_port</td>
<td>This feature indicates the support of RDS dynamic port management.</td>
</tr>
<tr class="even">
<td>7</td>
<td>Rds_serialization_format</td>
<td>This feature indicates the support of RDS Serialization Format reservation, notification, and query. This feature requires that the Rds_dynamic_port is also supported.</td>
</tr>
<tr class="odd">
<td>8</td>
<td>PatchUpdate</td>
<td>Indicates the support of enhancements to the northbound interfaces (e.g. support the partial modification of an existing subscription resource).</td>
</tr>
