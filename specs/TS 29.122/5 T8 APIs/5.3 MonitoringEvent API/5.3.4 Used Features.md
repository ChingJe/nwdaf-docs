# 5.3.4 Used Features

The table below defines the features applicable to the MonitoringEvent API. Those features are negotiated as described in clause 5.2.7.

Table 5.3.4-1: Features used by MonitoringEvent API

</tr>
<tr class="odd">

<td>Ue-reachability_notification</td>

<td>Number_of_UEs_in_an_area_notification</td>

<p>The feature supports pre-5G (e.g. 4G) requirement.</p></td>
</tr>
<tr class="even">
<td>9</td>
<td>Notification_websocket</td>

</tr>
<tr class="odd">
<td>10</td>
<td>Notification_test_event</td>
<td>The testing of notification connection is supported according to clause 5.2.5.3.</td>
</tr>
<tr class="even">
<td>11</td>
<td>Subscription_modification</td>
<td>Modifications of an individual subscription resource.</td>
</tr>
<tr class="odd">
<td>12</td>
<td>Number_of_UEs_in_an_area_notification_5G</td>
<td><p>The AF is notified the number of UEs present in a given geographic area.</p>
<p>The feature supports the 5G requirement. This feature may only be supported in 5G.</p></td>
</tr>
<tr class="even">
<td>13</td>
<td>Pdn_connectivity_status</td>
<td>The SCS/AS requests to be notified when the 3GPP network detects that the UE’s PDN connection is set up or torn down.</td>
</tr>
<tr class="odd">
<td>14</td>
<td>Downlink_data_delivery_status_5G</td>
<td>The AF requests to be notified when the 3GPP network detects that the downlink data delivery status is changed. The feature is not applicable to pre-5G.</td>
</tr>
<tr class="even">
<td>15</td>
<td>Availability_after_DDN_failure_notification_enhancement</td>
<td>The AF is notified when the UE has become available after a DDN failure and the traffic matches the packet filter provided by the AF. The feature is not applicable to pre-5G.</td>
</tr>
<tr class="odd">
<td>16</td>
<td>Enhanced_param_config</td>
<td>This feature supports the co-existence of multiple event configurations for target UE(s) if there are parameters affecting periodic RAU/TAU timer and/or Active Time. Supporting this feature also requires the support of feature number 1 or 2.</td>
</tr>
<tr class="even">
<td>17</td>
<td>API_support_capability_notification</td>
<td>The SCS/AS is notified of the availability of support of service APIs. This feature is only applicable in interworking SCEF+NEF scenario.</td>
</tr>
<tr class="odd">
<td>18</td>
<td>eLCS</td>
<td><p>This feature supports the enhanced location exposure service (e.g. location information preciser than cell level).</p>
<p>The feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="even">
<td>19</td>
<td>NSAC</td>
<td><p>This feature controls the support of the Network Slice Admission Control (NSAC) functionalities.</p>
<p>The feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="odd">
<td>20</td>
<td>Partial_group_modification</td>
<td>This feature supports the partial cancellation and/or partial addition to the group member(s) within the grouped event monitoring subscription.</td>
</tr>
<tr class="even">
<td>21</td>
<td>UAV</td>
<td><p>The SCS/AS requests to be notified of the UAV presence status in a specific geographic area. This feature is only applicable in interworking SCEF+NEF scenario, or standalone 5G scenario.</p>
<p>This feature requires that Number_of_UEs_in_an_area_notification and Number_of_UEs_in_an_area_notification_5G features are also supported.</p></td>
</tr>
<tr class="odd">
<td>22</td>
<td>MULTIQOS</td>
<td><p>This feature indicates the support for "Multiple QoS Class" which enables to support more than one Location QoS during LCS procedures.</p>
<p>This feature requires that the eLCS feature is also supported.</p></td>
</tr>
<tr class="even">
<td>23</td>
<td>Session_Management_Enhancement</td>
<td>This feature supports Session Management enhancement with requested DNN and/or S-NSSAI. This feature requires that the Pdn_connectivity_status feature or Downlink_data_delivery_status_5G feature is also supported.</td>
</tr>
<tr class="odd">
<td>24</td>
<td>enNB</td>
<td>Indicates the support of enhancements to the northbound interfaces.</td>
</tr>
<tr class="even">
<td>25</td>
<td>EDGEAPP</td>
<td><p>This feature controls the support of EDGE applications related functionalities (e.g. support the civic address as a possible location granularity).</p>
<p>The feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="odd">
<td>26</td>
<td>UEId_retrieval</td>
<td>This feature supports AF specific UE ID retrieval which is not applicable to pre-5G (e.g. 4G).</td>
</tr>
<tr class="even">
<td>27</td>
<td>UserConsentRevocation</td>
<td>This feature indicates the support of user consent revocation management and enforcement (e.g. stop data processing) for EDGE applications.</td>
</tr>
<tr class="odd">
<td>28</td>
<td>Subscription_Patch</td>
<td>This feature indicates the support of the PATCH method for partial modification of an existing event monitoring subscription.</td>
</tr>
<tr class="even">
<td>29</td>
<td>GMEC</td>
<td><p>This feature indicates the support of Generic Group Management, Exposure and Communication Enhancements.</p>
<p>The following functionalities are supported:</p>
<p>- Support Group Members List Change event reporting.</p>
<p>This feature is not applicable to pre-5G (e.g., 4G).</p></td>
</tr>
<tr class="odd">
<td>30</td>
<td>Loss_of_connectivity_notification_5G</td>
<td><p>The AF is notified when the 3GPP network detects that the UE is no longer reachable for signalling or user plane communication.</p>
<p>This feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="even">
<td>31</td>
<td>enNB1</td>
<td>Indicates the support of enhancements to this northbound API in Rel-18.</td>
</tr>
<tr class="odd">
<td>32</td>
<td>AppDetection_5G</td>
<td><p>This feature indicates the support of Application traffic detection (e.g., start and stop) monitoring event.</p>
<p>This feature is not applicable to pre-5G (e.g., 4G).</p></td>
</tr>
<tr class="even">
<td>33</td>
<td>enNB1_5G</td>
<td><p>Indicates the support of enhancements to this northbound API for 5G in Rel-18.</p>
<p>This feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="odd">
<td>34</td>
<td>eLCS_en</td>
<td><p>This feature indicates the support of the enhancements to the eLCS feature.</p>
<p>The following functionalities are supported:</p>
<p>- Support the error handling related to the area event reporting for the case where the requested location area is not allowed.</p>
<p>- Support location reporting over user plane between UE and AF.</p>
<p>This feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="even">
<td>35</td>
<td>eNSAC</td>
<td><p>This feature indicates the support of the enhancements to the NSAC feature.</p>
<p>The following functionalities are supported:</p>
<p>- Support the status notification of the current number of UEs with at least one PDU session/PDN connection.This feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="odd">
<td>36</td>
<td>Ranging_SL</td>
<td><p>This feature indicates the support of the ranging and sidelink positioning functionality.</p>
<p>The following functionalities are supported:</p>
<p>- Support the ranging and sidelink input/output parameters.</p>
<p>This feature requires the support of eLCS feature.</p>
<p>This feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
<tr class="even">
<td>37</td>
<td>DataTransfer</td>
<td><p>This feature indicates the support of Session inactivity time, Traffic volume and UL/DL data rate events for data transfer and measurement.</p>
<p>This feature is not applicable to pre-5G (e.g. 4G).</p></td>
</tr>
